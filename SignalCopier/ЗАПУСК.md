# Cryptonus_Trade Signal Copier — Порядок запуска

---

## Шаг 1 — Установка зависимостей

```bash
pip install telethon ccxt python-dotenv
```

Один раз, больше не нужно.

---

## Шаг 2 — Получить Telegram API ключи

1. Зайти на **https://my.telegram.org**
2. Войти под своим номером телефона
3. → "API development tools"
4. Создать приложение (название любое, например "signal_copier")
5. Скопировать **App api_id** и **App api_hash**

---

## Шаг 3 — Создать OKX API ключ

1. Зайти в OKX → Профиль → API
2. Создать новый ключ с правами: **Read + Trade**
3. Без права вывода (не нужно и небезопасно)
4. Сохранить: **API Key**, **Secret Key**, **Passphrase**

---

## Шаг 4 — Настроить .env

Скопировать `.env.example` в `.env` и заполнить:

```
TG_API_ID=12345678
TG_API_HASH=abcdef1234567890abcdef1234567890

OKX_API_KEY=ваш_ключ
OKX_SECRET=ваш_секрет
OKX_PASSPHRASE=ваша_фраза

OKX_TESTNET=true      # ← сначала true, потом поменять на false
RISK_USDT=10          # риск на сделку в USDT
```

---

## Шаг 5 — Авторизация в Telegram (один раз)

```bash
python signal_copier.py
```

При первом запуске скрипт спросит:

```
Please enter your phone (or bot token): +7XXXXXXXXXX
Please enter the code you received: 12345
```

После ввода кода создастся файл `cryptonus_session.session`.
Следующие запуски — без вопросов, авторизация автоматическая.

**Остановить:** Ctrl+C

---

## Шаг 6 — Проверка на тестнете OKX

В `.env` должно быть `OKX_TESTNET=true`.

Запустить скрипт, дождаться сигнала из канала.
В логах должно появиться:

```
MSG [NEW_SIGNAL] received:
#RLCUSDT 15m STATUS: LONG ...

TimingGate: offset=12s / 300s → PASS
Signal parsed: Trade(symbol='RLCUSDT' ...)
ENTRY1 placed: 45 @ 0.3681  id=xxx
ENTRY2 placed: 45 @ 0.346   id=xxx
TP1 placed: 15 @ 0.3755     id=xxx
...
```

Проверить в OKX (Testnet) что ордера появились.

---

## Шаг 7 — Включить боевой режим

В `.env` изменить:

```
OKX_TESTNET=false
```

Перезапустить скрипт. Всё — работает в реале.

---

## Шаг 8 — Анализ истории (по желанию, в любое время)

### Скачать историю и проанализировать:

```bash
python analyze_history.py --days 60
```

Скачает историю канала за 60 дней, сохранит в `history_cache.json`,
выведет рейтинг стратегий выхода и per-symbol статистику.

### Повторный анализ без скачивания (быстро, из кэша):

```bash
python analyze_history.py --no-fetch
```

### Детальный разбор по символу:

```bash
python analyze_history.py --no-fetch --symbol BTC
python analyze_history.py --no-fetch --symbol RLCUSDT
```

## Структура файлов после запуска

```
signal_copier.py          ← основной скрипт (запускать постоянно)
analyze_history.py        ← анализ истории (запускать по желанию)
.env                      ← ваши ключи (не светить нигде)
cryptonus_session.session ← Telegram сессия (не удалять)
okx_symbols.json          ← кэш символов OKX (обновляется раз в неделю)
history_cache.json        ← кэш истории канала (после analyze_history)
signal_copier.log         ← лог всех событий
```

---

## Что происходит в фоне пока скрипт работает

| Событие | Действие |
|---------|----------|
| Новое сообщение в канале | Классифицируется и логируется |
| Тип UPDATE/TP_HIT/RESULT | Тихо игнорируется |
| Новый сигнал вне M5 окна | Пропускается, пишется в лог |
| Новый сигнал в M5 окне | Парсится и исполняется на OKX |
| Символ не найден на OKX | Пропускается, пишется в лог |
| Раз в неделю | Список символов OKX обновляется |

---

## Запуск в фоне (Linux/Mac)

```bash
nohup python signal_copier.py > signal_copier.log 2>&1 &
echo $! > copier.pid
```

Остановить:
```bash
kill $(cat copier.pid)
```

---

## Частые вопросы

**Потерял сессию / нужно перелогиниться:**
```bash
rm cryptonus_session.session
python signal_copier.py   # снова попросит код
```

**Форсировать обновление символов OKX:**
```bash
rm okx_symbols.json
# перезапустить скрипт — обновится автоматически
```

**Посмотреть лог в реальном времени:**
```bash
tail -f signal_copier.log
```
